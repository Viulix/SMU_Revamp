# E5263A I–V Measurement — Implementation Notes

This file summarizes how the project drives an E5263A-like SMU to perform I–V measurements (voltage sweeps and single-point forced measurements). The text is extracted from `voltage_sweep.vb`, `meas.vb` and related code (`switchmatrix.vb`, `prober.vb`). Use this to implement an instrument driver and measurement routine.

---

## Overview
- Instrument resource is opened via VISA (ResourceManager.Open).
- In this code the E5263A flow is triggered when `E5263A_2ch.Checked` is true; the session used is opened at `GPIB0::17::INSTR`.
- Measurement flow includes: configure SMU channels, define waveform (WV) for voltage sweep, set ranges/averaging, start measurement (TSR/XE/TSQ), read binary/ASCII response and parse values.
- Before a measurement ensure the prober is at the contact (`move_prober_absolute`) and the switch matrix is configured (`create_connection`).

## Typical GPIB sequence (high level)
1. Open session: `session = ResourceManager.Open("GPIB0::17::INSTR")`
2. Configure and reset: `*RST`
3. Select/enable channel(s): `CN <n>` (example: `CN 2`)
4. (Optional) set DC voltage/current forcing for channel: `DV` or `DI` depending on mode
   - `DV <ch>,<range>,<start>,<Icomp>` (example usage in code: `DV 2,20,0,0.1`)
   - `DI <ch>,<range>,<forceV>,<vcompliance>` used when forcing voltage and measuring current in `meas.vb` functions
5. Set format/measurement options:
   - `FMT 1` — set data format
   - `TSC 1` — enable timestamps
   - `FL 0` or `FL 1` — filter off/on
   - `AV <n>,0` — ADC averaging/PCLKs (negative value used to indicate PLCs in code `"-" & adc_samples`)
6. Define waveform (sweep): `WV <ch>,<mode>,<range>,<start>,<stop>,<steps>,<Icomp>`
   - Example from code:
     `WV <channel>,3,0,0,<V>,<NoOfPoints>,<Icc>`
     - mode `3` used = double staircase (start→stop→start)
     - range `0` = auto
     - `<V>` = end voltage
     - `<NoOfPoints>` = number of points
     - `<Icc>` = compliance/current limit
7. Set measurement and conversion:
   - `RI <ch>,0` — range auto
   - `MM 2,<ch>` — measurement mode (e.g., staircase)
   - `CMM <ch>,1` — choose measurement quantity (1 = current measurement)
8. Trigger/collect data:
   - `TSR`
   - `XE`
   - `TSQ`
   - Optionally `CL <ch>` to clear buffers after
9. Read result:
   - `mret = session.ReadString(<expected_length>)`
   - `tret = session.ReadString(17)` for timestamp
   - Parse: timestamp/calibration extracted via `Mid(mret,4,12)` etc. Example parsing in VB:
     - `tcal = Mid(mret, 4, 12)`
     - `tret = Mid(tret, 4, 12)`
     - `mtime = Val(tret) - Val(tcal)`
     - `status = Mid(mret, 17, 3)`
     - measurement value: `Val(Mid(mret, 20, 12))`
10. Close session

## Single-point forced measurements (examples from `meas.vb`)
- Force voltage, measure current (`force_i_meas_v`):
  - `DI <ch>,<range>,<forceV>,<vcompliance>`
  - `MM 1,<ch>` (spot measurement)
  - `CMM <ch>,2` or `CMM <ch>,1` depending on required measured quantity
  - `RV <ch>,` query for result and then parse returned string
  - `TSR`, `XE`, `TSQ` used similarly

- Force current, measure voltage (`force_v_meas_i`):
  - `DV <ch>,<range>,<forceI>,<vcompliance>` or similar
  - set `AV` samples and `FL` filter
  - `MM 1,<ch>` and `CMM <ch>,1` for current measurement
  - Use `RI` to control range or auto-range

## Notes on parsing returned data
- Code reads large blocks with `ReadString(n)` and then parses fixed-width substrings.
- Example parse pattern used repeatedly:
  - `tcal = Mid(mret,4,12)` — calibration/time base
  - `tret = Mid(tret,4,12)` — timestamp
  - measurement value e.g. `Val(Mid(mret,20,12))` (12-character field)
  - status bits at `Mid(mret,17,3)`
- Implementer must inspect the exact SCPI-like format returned by this SMU or test with `Query("TSQ?"/other)`, adjust parse offsets accordingly.

## Switchmatrix & prober integration (short)
- Switch matrix is controlled via `GPIB0::23::INSTR` (`switchmatrix.vb`) using commands:
  - `:ROUT:CONN:RULE ALL,FREE`
  - `:ROUT:CONN:SEQ ALL,BBM`
  - `:ROUT:CLOSE (@<path>)` where `<path>` is like `@1xx, 1yy` or `@xxxx,yy` depending on formatting
- Prober movement uses `GPIB0::22::INSTR` and commands like `MoveChuck <x> <y> H` / `MoveChuckZ <z> R` and `EnableMotorQuiet 1`.
- Ensure you call `create_connection(pathX, pathY)` to route the needle to the SMU before starting the SMU measurement, and `move_prober_absolute(-deltax, deltay)` to position needles.

## Example C# pseudocode (instrument driver + sweep)
```csharp
// open
var rm = new ResourceManager();
var session = rm.Open("GPIB0::17::INSTR");
session.Timeout = 300000;

// reset and base config
session.Write("*RST\n");
session.Write("FMT 1\n");
session.Write("TSC 1\n");

// configure channel
session.Write($"CN {channel}\n");
session.Write($"AV -{adcSamples},0\n"); // use negative to indicate PLCs as in original code

// define sweep (double staircase example)
session.Write($"WV {channel},3,0,0,{V_end},{nPoints},{I_comp}\n");
session.Write($"RI {channel},0\n");
session.Write($"MM 2,{channel}\n");
session.Write($"CMM {channel},1\n");

// trigger
session.Write("TSR\n");
session.Write("XE\n");
session.Write("TSQ\n");

// read result
string raw = session.ReadString(expectedLength);
// parse raw as fixed-width fields (follow project's parsing offsets)

// finish
session.Write($"CL {channel}\n");
session.Close();
```

## Implementation tips
- Mirror the VB parse offsets initially; test with hardware and adjust expected read length and field boundaries.
- Use the same averaging (AV) and filter (FL) semantics as in the VB code when porting results to match.
- Respect the switchmatrix and prober steps: configure matrix path, move prober into contact, wait for stabilization, then perform SMU commands.
- Add robust error handling/timeouts for GPIB timeouts and `ReadString` sizes.
